# โจทย์ที่ 8: Business logic & Summary (5 คะแนน)

## ข้อ 8.1 – `GET /purchases/summary` (2 คะแนน)

สรุปภาพรวมคำสั่งซื้อทั้งหมด

**Path:** `GET /purchases/summary`

**ไฟล์ที่ต้องแก้ไข:**

- `src/purchases/purchase.service.ts` (เพิ่ม method `getSummary()`)
- `src/purchases/purchase.controller.ts` (เพิ่ม endpoint `GET /purchases/summary`)

**เงื่อนไข:**

- `totalPurchases` = จำนวนคำสั่งซื้อทั้งหมด
- `totalRevenue` = ผลรวม `totalPrice` ของทุกคำสั่งซื้อ
- `averagePurchaseValue` = `totalRevenue / totalPurchases` (ปัดเศษเป็นจำนวนเต็ม)
- `totalProductsSold` = ผลรวม `quantity` ของทุก items ในทุกคำสั่งซื้อ
- Response ต้องเป็นรูปแบบ `ApiResponse<T>`

**ตัวอย่าง Response:**

```json
{
  "success": true,
  "data": {
    "totalPurchases": 5,
    "totalRevenue": 141200,
    "averagePurchaseValue": 28240,
    "totalProductsSold": 73
  },
  "message": "Fetched purchases summary successfully"
}
```

## ข้อ 8.2 – `GET /products/sales-report` (3 คะแนน)

รายงานยอดขายของสินค้าทุกชิ้น

**Path:** `GET /products/sales-report`

**ไฟล์ที่ต้องแก้ไข:**

- `src/products/product.service.ts` หรือ `src/purchases/purchase.service.ts` (เพิ่ม method `getSalesReport()`)
- `src/products/product.controller.ts` (เพิ่ม endpoint `GET /products/sales-report`)

**เงื่อนไข:**

- ต้องอ่านข้อมูลจากทั้ง `data/products.json` และ `data/purchases.json`
- สรุปยอดขายแต่ละสินค้า: `totalSold` = ผลรวม `quantity` ที่ขายไป, `totalRevenue` = ผลรวม `quantity * price`
- Response ต้องเป็นรูปแบบ `ApiResponse<T>`

**ตัวอย่าง Response:**

```json
{
  "success": true,
  "data": [
    {
      "productId": 1,
      "productName": "Laptop",
      "totalSold": 3,
      "totalRevenue": 75000
    },
    {
      "productId": 2,
      "productName": "Mouse",
      "totalSold": 25,
      "totalRevenue": 12500
    },
    {
      "productId": 3,
      "productName": "Keyboard",
      "totalSold": 6,
      "totalRevenue": 7200
    }
  ],
  "message": "Fetched product sales report successfully"
}
```

**ข้อห้าม:**

1. ห้ามใช้ `process.exit()` มิเช่นนั้นจะตัดคะแนนเป็น 0 ทันที
2. ห้ามแก้ไข file `products.json` หรือ `purchases.json` หากใช้เพื่อทดสอบเสร็จแล้วให้แก้ไขเป็นข้อมูลเดิมหรืออย่า push ขึ้น git repository.
