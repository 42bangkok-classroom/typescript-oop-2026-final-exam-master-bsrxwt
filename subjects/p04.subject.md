# โจทย์ที่ 4: GET purchase by id (2 คะแนน)

ดึงข้อมูลคำสั่งซื้อตาม `id` และจัดการกรณีไม่พบ

**Path:** `GET /purchases/:id`

**ไฟล์ที่ต้องแก้ไข (จากข้อ 2):**

- `src/purchases/purchase.service.ts` (เพิ่ม method `findOne(id: number)`)
- `src/purchases/purchase.controller.ts` (เพิ่ม endpoint `GET /purchases/:id`)

**เงื่อนไข:**

- อ่านข้อมูลจาก `data/purchases.json` เพื่อหาคำสั่งซื้อที่มี `id` ตรงกัน
- Response ต้องเป็นรูปแบบ `ApiResponse<T>`

### กรณีพบข้อมูล (1 คะแนน)

**Request:**

```text
GET /purchases/1
```

**Response:**

```json
{
  "success": true,
  "data": {
    "id": 1,
    "customerName": "John Doe",
    "purchaseDate": "2024-01-15",
    "items": [
      { "productId": 1, "quantity": 2, "price": 25000 },
      { "productId": 2, "quantity": 5, "price": 500 }
    ],
    "totalPrice": 52500
  },
  "message": "Fetched purchase successfully"
}
```

### กรณีไม่พบข้อมูล (1 คะแนน)

**Request:**

```text
GET /purchases/999
```

**Response:**

```json
{
  "success": false,
  "data": null,
  "message": "Purchase with id 999 not found"
}
```

**ข้อห้าม:**

1. ห้ามใช้ `process.exit()` มิเช่นนั้นจะตัดคะแนนเป็น 0 ทันที
2. ห้ามแก้ไข file `products.json` หรือ `purchases.json` หากใช้เพื่อทดสอบเสร็จแล้วให้แก้ไขเป็นข้อมูลเดิมหรืออย่า push ขึ้น git repository.
